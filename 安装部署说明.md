# 安装部署说明（开源版 / AppAgent_mobile）

## 1. 运行形态

- **开发机**：Windows（推荐 PowerShell 7）
- **构建环境**：WSL（Ubuntu）+ buildozer + python-for-android
- **运行环境**：Android 真机/模拟器（开启 USB 调试）
- **设备交互**：ADB（截图、XML、输入事件）

## 2. 必备条件

### 2.1 Windows 侧

- 安装 **ADB / platform-tools**，并确保 `adb` 在 PATH 中
- 安装 **WSL**（Ubuntu），可执行 `wsl.exe -e bash -lc "echo ok"`

### 2.2 Android 侧

- 开启开发者选项与 USB 调试
- 连接电脑后执行：

```powershell
adb devices
```

看到目标设备处于 `device` 状态即可（`unauthorized` 需要在手机上点允许）。

## 3. 配置（非常重要）

### 3.1 配置多模态模型 Key

配置文件：[config.yaml](file:///g:/AppAgent/AppAgent_mobile_bak/config.yaml)

- `OPENAI_API_KEY` / `DASHSCOPE_API_KEY` / `ARK_API_KEY`
- 不配置 key 时，执行器会提示未配置并跳过/失败（取决于运行路径）

### 3.2 ADB 目录配置

`config.yaml` 中：

- `ANDROID_SCREENSHOT_DIR`：设备侧保存截图的目录（默认 `/sdcard/AppAgent/screenshots`）
- `ANDROID_XML_DIR`：设备侧保存 XML 的目录（默认 `/sdcard/AppAgent/xml`）

一般不需要改；如果你改了，请确保目录存在（脚本会尝试 `mkdir -p`，但某些 ROM 权限策略不同）。

## 4. 构建与安装（推荐方式）

项目根目录：`G:\AppAgent\AppAgent_mobile_bak`

### 4.1 一键构建 + 安装（推荐）

```powershell
.\one_click_build_install.ps1
```

- 会在 WSL 内执行 [build_apk.sh](file:///g:/AppAgent/AppAgent_mobile_bak/build_apk.sh) 完整构建
- 构建完成后 APK 会复制回 Windows 项目根目录
- 然后自动执行 `adb install -r` 安装到目标设备

如果你电脑连接了多台设备，请手动指定：

```powershell
.\one_click_build_install.ps1 -Serial <adb_device_serial>
```

如果你想强制先卸载再安装（更干净）：

```powershell
.\one_click_build_install.ps1 -ForceUninstall
```

### 4.2 快速构建（开发迭代）

```powershell
.\quick_build.ps1 -Install
```

遇到构建缓存/奇怪错误时：

```powershell
.\quick_build.ps1 -Clean -Install
```

如果遇到类似下面的报错：
- `ModuleNotFoundError: No module named 'setuptools'`（通常发生在 Pillow 编译阶段）

解决方法：
- 确保 [buildozer.spec](file:///g:/AppAgent/AppAgent_mobile_bak/buildozer.spec) 的 `requirements` 包含 `setuptools,wheel`
- 然后重新执行一次完整构建：

```powershell
.\one_click_build_install.ps1
```

## 5. 运行与验证

1. 手机上启动应用（包名：`org.test.orderquery`）
2. 在底部输入框输入任务文本，点击发送（或侧边栏 Run Task）
3. 观察任务产物目录（应用私有目录）：
   - 任务目录（推荐）：`/data/user/0/org.test.orderquery/files/app/tasks/`
   - 任务目录（部分旧版本/不同实现可能使用）：`/data/user/0/org.test.orderquery/files/tasks/`

验证点：
- 每轮能生成 `*_N.png`、`*_N.xml`
- labeled 模式下生成 `*_N_labeled.png`（若 `cv2` 不存在会自动使用 PIL 兜底）
- 日志不应再出现 `No such file or directory: *_labeled.png`

## 6. 日志导出（定位问题用）

导出任务目录（示例）：

```powershell
adb shell run-as org.test.orderquery ls -l files/app/tasks
adb pull /data/user/0/org.test.orderquery/files/app/tasks .\\downloaded_tasks
```

如果电脑连接了多台设备，请追加 `-s <serial>`。

## 7. 文件精简建议（部署包/仓库）

以下内容建议不要长期保留在仓库中（运行时会自动生成）：

- `*.apk`、`bin/`、`downloaded_apk/`
- `logs/`、`tasks/`、`*.log`、调试截图（例如 `*output*.png`）

仓库已提供 `.gitignore`，默认会忽略上述产物文件。
